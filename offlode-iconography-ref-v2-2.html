Colour Rule
Icons inherit currentColor from their parent text colour. Never hardcode icon colours — they adapt to dark/light mode automatically. The only exceptions are agent icons (Collect/Respond/Settle) which use their dedicated agent colours, and status icons which use semantic colours.

Contextual Usage Guide

Where each icon button variant appears in the platform.

Header toolbar: Ghost filled 28px — search, notifications, theme toggle, profile
Table row actions: Ghost (no bg) 24px — edit, delete, more menu. Appear on hover.
Card header: Ghost filled 28px — more menu (⋯), close, expand
Workspace toggle: Ghost filled 28px — triage/focus mode switch (sidebar/layout icon)
List collapse: Ghost (no bg) 28px — chevron toggle for client list column
Dialog close: Outlined 28px — modal/panel close button (✕)
Floating action: Brand filled 36px — primary add/create button
Destructive confirm: Danger 28px — delete confirmation in modals
Hover & Active States
Ghost filled: background opacity increases from 0.08 → 0.12 on hover, 0.15 on active. Outlined: border opacity increases from 0.10 → 0.16 on hover. Ghost (no bg): background appears at 0.06 on hover. Brand: darken 10% on hover. Danger: background opacity increases from 0.10 → 0.16 on hover. All transitions use 150ms ease.
